# v2.0.0 Release Summary

## Completed

- Named `FieldState` blocks with stable field size and offset queries.
- Callback-based `PhysicsOperator` contract with no deep inheritance hierarchy.
- `CoupledProblem` aggregation for coupled matrix/residual assembly.
- Magnetostatic operator adapter that embeds a magnetic block and computes `K A - f`.
- Canonical magneto-thermal heat operator with thermal residual and magnetic-to-thermal cross-Jacobian entries.
- Version-readiness analysis for `v1.0` through `v2.0`.
- Inheritance-depth analysis remains within the `<= 3` style rule.

## Not Completed

- Production thermal conduction.
- Production magneto-mechanical coupling.
- Shared time integration policy.
- Global nonlinear coupled Newton solve.
- Routing the full machine `motor_solve` path through `CoupledProblem`.

## Known Risks

- The canonical magneto-thermal operator proves coupling shape and Jacobian plumbing, but it is not a physical thermal solver.

## Compatibility Changes

- New public coupling APIs under `include/femsolver/coupling`.
- Project version is advanced to `2.0.0-dev` to mark the completed foundation chain, not an industrial product release.

## Next Version Carryover

- Connect nonlinear B-H cell residuals to the global magnetostatic solve.
- Route machine solves through the coupled operator stack.
- Add real thermal conduction and boundary-condition models.
