# `v0.2.0` Draft Release Summary

## Completed

- Introduced a self-owned `kernel/` module tree with explicit boundaries for `mesh`, `reference`, `quadrature`, `basis`, `algebra`, `assembly`, and `benchmark`.
- Implemented a tetrahedral mesh container with validation, boundary-face storage, boundary-node discovery, and orientation normalization.
- Implemented the reference tetrahedron, a centroid quadrature rule, and a linear `H1` basis.
- Implemented element geometry mapping, scalar Poisson local/global assembly, Dirichlet elimination, and a minimal conjugate-gradient solver.
- Added a manufactured Poisson benchmark and wired it into `motor_check`.
- Added technical documentation, ADR coverage, and unit/integration tests for the full `v0.2.0` path.

## Not Completed

- `H(curl)` basis functions and vector assembly
- electromagnetic source terms
- nonlinear material behavior
- industrial mesh import and preprocessing
- multiphysics coupling

## Known Risks

- the current quadrature and algebra layers are intentionally minimal and will need extension before real electromagnetics.
- Dirichlet elimination is explicit and auditable, but not yet optimized for large systems.
- application-level solver/report types still include placeholders while the new kernel is growing beneath them.

## Compatibility Changes

- The repository now contains a real self-owned kernel API under `include/femsolver/kernel/...`.
- `motor_check` now executes the canonical Poisson benchmark by default instead of only printing a placeholder summary.

## Next Version Handoff

- Preserve the new kernel boundaries.
- Extend the kernel to edge-based topology and `H(curl)` vector elements in `v0.3.0`.
- Keep the rule that each meaningful kernel increment must ship with technical docs and tests before it is considered complete.
