# v1.0.0 Release Summary

## Completed

- Linear robot-joint motor magnetostatic smoke path using tetrahedral first-order Nedelec elements.
- Case and mesh-manifest driven `motor_solve`.
- Current-density and permanent-magnet source contributions.
- Cell-wise `B = curl(A)`, magnetic energy, preliminary torque-surface summary, and regression text output.
- `motor_check --machine-regression` for `joint_type_i_12s10p`.

## Not Completed

- Nonlinear magnetic material assembly.
- Faithful slot/tooth/winding/magnet geometry reconstruction.
- Validated torque accuracy on real machine meshes.
- Periodic sectors, motion, eddy currents, thermal, and circuit coupling.

## Known Risks

- Simplified geometry and excitation reductions make early machine outputs regression signals, not validated design numbers.
- Gmsh availability affects whether the imported mesh path or synthetic fallback path runs locally.

## Compatibility Changes

- `motor_solve` now accepts `--case` and `--manifest`.
- `motor_check` includes a machine regression mode.

## Next Version Carryover

- Add nonlinear B-H material laws and tangent validation in `v1.1.0`.
