# `v0.3.0` Draft Release Summary

## Completed So Far

- Added a tracked machine-case catalog for robot-joint motors and one industrial envelope case.
- Added a simplified preprocessing path that validates case schema, generates `.geo` files, optionally invokes `gmsh`, and emits manifests plus summaries.
- Extended the self-owned tetrahedral kernel from scalar `H1` toward vector `H(curl)` support with edge topology, edge DoF maps, first-order `Nedelec` basis functions, and curl-curl assembly.
- Added a canonical tetrahedral `H(curl)` benchmark and exposed it through `motor_check`.
- Added native C++ loading for the current case and generated manifest schema.
- Added a linear magnetostatic smoke path in `motor_solve` with current-density and remanent-flux source support.
- Added the first Gmsh `MSH2` tetra import path, field-summary recovery, air-gap torque-surface metadata, and machine regression gate.
- Added technical documentation and automated tests for both the case-preprocessing path and the vector-kernel baseline.

## Not Completed

- Detailed tooth, slot, wedge, and magnet geometry reconstruction
- General-purpose mesh import beyond the current Gmsh `MSH2` tetra bridge
- `H(div)` support, hexahedral support, and higher-order support
- Machine-level validation against published torque, flux, or overload targets

## Known Risks

- `motor_pre` still bridges into Python tooling instead of using a fully native C++ path.
- The generated geometries are simplified concentric envelopes and must not be confused with faithful paper geometry.
- `motor_solve` imports generated meshes when available, but still falls back to the synthetic region-tagged smoke mesh when preprocessing has no usable `.msh`.
- The current torque estimate is a regression signal on the simplified air-gap surface, not a validated motor-performance value.

## Compatibility Changes

- `motor_pre` now accepts real `--case`, `--variant`, and `--output-dir` arguments.
- `motor_check` now includes both scalar and vector canonical benchmark coverage.
- `motor_check` now includes the first `--machine-regression` gate.
- Machine cases now declare the intended production space family as `nedelec`.
- `motor_solve` now accepts `--case` and `--manifest` for the first linear joint-motor smoke path.

## Next Version Handoff

- Preserve the new `kernel/space` ownership boundary and keep higher-order, hex, and `RT` paths above the same interface contracts.
- Upgrade `joint_type_i_12s10p` from concentric-envelope geometry toward the first tooth, slot, winding, and magnet reconstruction.
- Tighten scalar machine regression thresholds once Gmsh-backed meshes are stable in CI.
