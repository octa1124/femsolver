# `v0.2.0` Kernel Foundation Overview

## Purpose

This document records what `v0.2.0` actually implements in code, what remains out of scope, and how the first self-owned FEM kernel executes from mesh input to benchmark result.

## Completed In `v0.2.0`

- a self-owned tetrahedral mesh container with validation and orientation normalization
- a reference tetrahedron definition
- a centroid quadrature rule on the reference tetrahedron
- a linear `H1` basis on the reference tetrahedron
- element geometry mapping and physical gradient recovery
- scalar Poisson local assembly and global sparse assembly
- Dirichlet boundary elimination on a mutable global system
- a minimal self-owned conjugate-gradient linear solve
- a manufactured linear Poisson benchmark exposed through `motor_check`
- unit tests and CLI coverage for the full path above

## Explicitly Not Completed In `v0.2.0`

- `H(curl)` / edge-element infrastructure
- electromagnetic weak forms
- material models beyond a scalar Poisson coefficient-free benchmark
- nonlinear residual and tangent assembly
- industrial mesh import
- multiphysics coupling

## End-To-End Execution Path

The implemented execution path is:

1. construct or load a tetrahedral mesh
2. normalize element orientation and validate connectivity
3. define a reference-cell quadrature rule and `P1` basis
4. build physical cell Jacobians and physical gradients
5. assemble local stiffness and load contributions
6. scatter local contributions into a mutable sparse global system
7. apply Dirichlet elimination to the assembled system
8. finalize the sparse matrix
9. solve the system with self-owned conjugate gradient
10. compare nodal values to the manufactured exact solution

## Why This Milestone Is Small On Purpose

The project needs a trustworthy kernel before it needs ambitious physics. `v0.2.0` therefore focuses on one scalar, auditable path that lets us test:

- topology ownership
- reference-to-physical mapping
- basis correctness
- local-to-global assembly contracts
- boundary-condition handling
- linear-solver plumbing

If any of these are unstable, adding `H(curl)` or motor physics on top would only hide problems instead of solving them.

## Hand-Off To `v0.3.0`

The next kernel step should preserve the current boundaries and add:

- edge topology and orientation
- first-kind Nedelec basis on tetrahedra
- curl-curl assembly
- canonical vector-field validation problems

`v0.3.0` should extend the kernel, not bypass it.
