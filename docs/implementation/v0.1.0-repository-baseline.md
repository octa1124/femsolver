# `v0.1.0` Repository Baseline

## Goal

`v0.1.0` establishes a repository baseline that is safe to extend into a self-owned solver kernel. The target is not solver physics yet; it is a reproducible engineering environment with governance, code layout, tests, release hygiene, and automation.

## Implemented Workstreams

- governance and documentation baseline
- CMake/CTest build bootstrap
- minimal application skeleton for `motor_pre`, `motor_solve`, and `motor_check`
- unit and Python tooling tests
- GitHub CI, release workflow, and bot configuration
- declarative repository-management metadata for labels, milestones, and branch rules

## Design Principle

The baseline is intentionally split so future solver work can grow without mixing concerns:

- governance files define rules and release behavior
- `src/` and `include/` hold product code only
- `tools/` holds repository and validation scripts
- `docs/implementation/` records technical decisions by workstream
- `.github/` holds repository automation and operational metadata

## Deliverables

- repository-wide rules in `Agent.md`
- changelog and release-note flow in `CHANGELOG.md`, `changes/unreleased/`, and `docs/releases/`
- build entrypoints and presets in `CMakeLists.txt` and `CMakePresets.json`
- a minimal core library and application executables
- automated tests and CI workflows

## Explicitly Deferred

- self-owned kernel implementation
- Gmsh geometry generation
- real case schema parsing
- nonlinear constitutive modeling
- robot-joint motor benchmark cases
