# `v1.2.0` Raviart-Thomas H(div) Implementation

## Scope

`v1.2.0` adds the first flux-conforming finite-element family to the self-owned kernel: lowest-order Raviart-Thomas on the reference tetrahedron.

## What It Can Do

- Evaluate the four `RT0` reference basis functions on a tetrahedron.
- Provide one face-oriented degree of freedom per tetrahedral face.
- Verify the face-flux interpolation property.
- Verify constant divergence and the divergence theorem on the reference element.
- Expose a canonical H(div) benchmark through `motor_check --hdiv-benchmark`.

## What It Cannot Do Yet

- It does not yet include a physical-to-reference H(div) Piola mapping.
- It does not yet include a global face DoF map across tetrahedral meshes.
- It does not yet assemble mixed Poisson, flux recovery, or post-processing systems.
- It does not yet support higher-order RT or hexahedral RT.

## Algorithm

On the reference tetrahedron with volume `1/6`, basis function `i` is:

```text
q_i(x) = (x - v_i) / (3 |K|)
```

where `v_i` is the vertex opposite face `i`. This gives:

- unit outward flux through face `i`
- zero flux through the other faces
- constant divergence `1 / |K| = 6`

The benchmark computes the face-flux matrix and divergence integral:

```text
integral(face_j, q_i dot n_j) = delta_ij
integral(K, div q_i) = 1
```

## Verification

The tests check:

- `RT0` face fluxes form an identity interpolation matrix
- every reference divergence equals `6`
- the canonical H(div) benchmark passes with tight tolerance
- the CLI benchmark is included in CTest

## Handoff

The next H(div) step is to add tetra face topology and a signed global face DoF map, then use the same RT basis for flux-conforming post-processing and mixed canonical problems.
